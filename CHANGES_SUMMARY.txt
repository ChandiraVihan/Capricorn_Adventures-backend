================================================================================
                        SECURITY FEATURES
================================================================================

✅ REFERENCE ID GENERATION
   • Implementation: UUID.randomUUID().substring(0, 8).toUpperCase()
   • Format: BK-XXXXXXXX (8 alphanumeric characters)
   • Uniqueness: Database UNIQUE constraint
   • Predictability: IMPOSSIBLE (UUID-based, no pattern)
   • Collision Prevention: Statistically impossible

✅ AUTHENTICATION
   • Method: JWT tokens via @Authorization header
   • Integration: SecurityContextHolder.getContext()
   • Scope: All booking endpoints protected
   • Stateless: Token-based, no server-side sessions

✅ AUTHORIZATION (BOLA/IDOR PREVENTION)
   • GET /api/bookings/ref/{id}
     → Validates user owns booking
     → Returns 403 Forbidden if unauthorized
   • GET /api/bookings/history
     → Returns only authenticated user's bookings
     → Query filtered by current user ID
   • POST /api/bookings
     → Links booking to authenticated user from JWT

✅ ERROR HANDLING
   • 404 Not Found: Non-existent bookings
   • 403 Forbidden: Unauthorized access attempts
   • 400 Bad Request: Invalid input data
   • JSON response format with timestamp
   • Centralized via GlobalExceptionHandler
